# Pingy Codex Rules

Do not redesign UI.
Preserve layout, spacing, typography, and visual hierarchy.

Make changes as small diffs.

Prefer updating app.js for logic.
Only update style.css if strictly necessary.

When adding new pages or routes:
- Reuse existing components
- Match current styling primitives
- Do not introduce new design systems

Always include a brief checklist of files changed in PR descriptions.
